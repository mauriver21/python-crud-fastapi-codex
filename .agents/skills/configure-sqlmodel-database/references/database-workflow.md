# Database and migration workflow

## Public interfaces

- `get_settings()` returns cached typed database settings.
- `get_engine()` returns a cached SQLAlchemy engine.
- `get_session()` yields one SQLModel session per dependency invocation and closes it afterward.

Override `get_session` in FastAPI tests. Clear `get_settings.cache_clear()` and `get_engine.cache_clear()` when a test changes environment configuration.

## Environment files

Set `APP_ENV` before importing database configuration. Settings load `.env` followed by `.env.<APP_ENV>`, so the environment-specific file wins. Use a dedicated PostgreSQL database for integration tests.

## Alembic

Register each table module from `<package>.db.models.__init__` so importing that package populates `SQLModel.metadata`.

```bash
alembic revision --autogenerate -m "create widgets"
alembic upgrade head
alembic downgrade -1
alembic current
```

Review autogenerated types, constraints, indexes, defaults, and downgrade behavior before applying a revision.
